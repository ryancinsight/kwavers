# CFB-GBM Local Execution Sample

This directory contains a small same-patient CT-backed GBM sample for Chapter 25
transcranial FUS planning. It enables the GBM branch to exercise tumor and skull
acoustics from the same patient without downloading the full 208 GB CFB-GBM cohort.

## Source

- **CT and T1 MR**: RIRE patient 109 from the Retrospective Image Registration
  Evaluation Project (`data/rire_patient_109/`).
  Source: https://rire.insight-journal.org/download_data
- **Segmentation**: Synthetic GBM tumor mask generated in the RIRE CT coordinate
  frame by `generate_synthetic_gbm_segmentation.py`.

## Files

| File | Origin | Description |
|------|--------|-------------|
| `ct.nii.gz` | RIRE patient 109 | Non-contrast head CT, 512×512×41, 0.404×0.404×3.0 mm |
| `t1.nii.gz` | RIRE patient 109 MR-T1 | Co-registered T1-weighted MR, 256×256×52, 0.82×0.82×3.0 mm |
| `segmentation.nii.gz` | Synthetic (generated) | Binary GBM tumor mask in CT space, 512×512×41 |
| `generate_synthetic_gbm_segmentation.py` | Local | Deterministic script to regenerate the segmentation |

## Missing Modalities

- **T1Gd** (contrast-enhanced T1): Not available in the RIRE dataset. The planning
  module falls back to the T1-weighted MR for the modality bridge.
- **FLAIR**: Not available in the RIRE dataset. The planning module proceeds with
  the CT-space segmentation as the primary reference.
- **T2**: Available in `data/rire_patient_109/` but not copied here; add
  `patient_109_mr_t2.nii.gz` as `t2.nii.gz` if T2-based planning is needed.

## Segmentation Details

The tumor segmentation is synthetic — it was algorithmically generated by
`generate_synthetic_gbm_segmentation.py` and is **not a clinical tumor
annotation**. It mimics radiographic GBM features (irregular lobulated
morphology, ~30 mm diameter, central necrosis) placed in the brain parenchyma.

**Generation**: `python generate_synthetic_gbm_segmentation.py`
(Deterministic; seed 42.)

The segmentation is a binary mask (1 = tumor, 0 = background) in the CT NIfTI
coordinate frame with the same affine as `ct.nii.gz`.

## Scope

This sample supplies a low-friction, fully local execution path for Chapter 25
GBM BBB-opening subspot planning when a same-patient CT-backed GBM case with
real clinical annotations is unavailable. The CT provides skull acoustics
(sound speed, attenuation); the segmentation provides the tumor target; the T1
MR provides the modality-bridge reference volume.

## Verification

```bash
python -c "
import nibabel as nib
ct = nib.load('ct.nii.gz')
seg = nib.load('segmentation.nii.gz')
assert ct.shape == seg.shape, 'Shape mismatch'
assert (seg.get_fdata() > 0).sum() > 0, 'Empty segmentation'
print('OK')
"
```
