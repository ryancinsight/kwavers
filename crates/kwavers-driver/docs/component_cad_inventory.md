## HV Driver Board

The canonical stack HV shield is a 24-channel board:

- Three `HV7355K6-G` devices are instantiated.
- `U1` routes channels `TX_0..TX_7`.
- `U2` routes channels `TX_8..TX_15`.
- `U3` routes channels `TX_16..TX_23`.
- The HV7355 serial interfaces are daisy-chained as one 24-bit shift/latch chain:
  `BUS_SDI -> U1.SDI -> U1.SDO -> U2.SDI -> U2.SDO -> U3.SDI -> U3.SDO -> BUS_SDO`.
- `driver_manifest.kv` must contain exactly 24 `TX_*` nets.

Local exact CAD is present:

- `docs/cad_models/HV7355K6_G/KiCADv6/footprints.pretty/QFN56_8X8MC_MCH.kicad_mod`
- `docs/cad_models/HV7355K6_G/KiCADv6/2026-06-19_15-17-36.kicad_sym`

The canonical stack shield currently uses a recovered routed artifact with renderable package and
connector CAD. The 32-channel comparison tile generated by `examples/hv7355_32ch_tile.rs` uses four
8-channel HV7355 routing abstractions and four 8-channel output headers rather than the exact
24-channel Molex transducer connector topology. Its KiCad output now includes visible package bodies:
stock KiCad QFN-56 8×8 render bodies for U1-U4, stock 0402 capacitor bodies, and stock 2.54 mm pin
header bodies with pad-grid-derived offsets and vertical two-column pad order matching the stock
KiCad models. Those render bodies make the 3D view inspectable and aligned to the generated pad
grids, but do not close the exact-footprint gap: the downloaded
`HV7355K6-G` QFN56 footprint has 57 pads and still needs a real pin-map/routing migration before the
32-channel board can be called component-complete.

The previous transducer connector target, Molex `0430452000`, is a 20-position connector and is no
longer sufficient for a single 24-channel tile. The selected same-family replacement is Molex
`0430452400`, a 24-position Micro-Fit 3.0 right-angle header, with matching receptacle housing
`0430252400`. Its exact KiCad footprint and STEP are present locally under
`docs/cad_models/430452400/`.

## Completion Gate

Generated boards now emit `component_accuracy_hv.kv` and `component_accuracy_fpga.kv`. A board is
component-complete only when the corresponding `exact_complete=true`. The current full-driver
manifests in `output/full_driver/` report `exact_complete=true` for the FPGA controller board and
the 24-channel HV tile. This is footprint/symbol completeness; 3D-body completeness is checked
separately from the KiCad `(model ...)` blocks.
